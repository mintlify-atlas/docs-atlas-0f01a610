## Available Themes

This configuration includes **151 professionally crafted themes** for Alacritty, covering a wide range of color palettes from popular dark themes to vibrant light themes.

<Note>
  The current theme is **coolnight**. All themes are located in `~/.config/alacritty/themes/themes/`.
</Note>

## Switching Themes

To change your theme, modify the import line in `~/.config/alacritty/alacritty.toml`:

```toml alacritty.toml
general.import = [
    "~/.config/alacritty/themes/themes/YOUR_THEME_NAME.toml"
]
```

<Tabs>
  <Tab title="Example: Tokyo Night">
    ```toml
    general.import = [
        "~/.config/alacritty/themes/themes/tokyo_night.toml"
    ]
    ```
  </Tab>
  
  <Tab title="Example: Catppuccin">
    ```toml
    general.import = [
        "~/.config/alacritty/themes/themes/catppuccin_mocha.toml"
    ]
    ```
  </Tab>
  
  <Tab title="Example: Gruvbox">
    ```toml
    general.import = [
        "~/.config/alacritty/themes/themes/gruvbox_dark.toml"
    ]
    ```
  </Tab>
  
  <Tab title="Example: Dracula">
    ```toml
    general.import = [
        "~/.config/alacritty/themes/themes/dracula.toml"
    ]
    ```
  </Tab>
</Tabs>

Alacritty will automatically reload the configuration when you save the file.

## Theme Preview Script

<Note>
  Want to preview themes quickly? Create a simple script to cycle through themes:
</Note>

```bash preview-themes.sh
#!/bin/bash
# Preview Alacritty themes

THEMES_DIR="$HOME/.config/alacritty/themes/themes"
CONFIG="$HOME/.config/alacritty/alacritty.toml"

for theme in "$THEMES_DIR"/*.toml; do
    theme_name=$(basename "$theme")
    echo "Previewing: $theme_name"
    
    # Update config
    sed -i "s|themes/themes/.*\.toml|themes/themes/$theme_name|"
    
    # Wait for user input
    read -p "Press Enter for next theme, 'q' to quit, 's' to select: " choice
    
    if [ "$choice" = "q" ]; then
        break
    elif [ "$choice" = "s" ]; then
        echo "Selected: $theme_name"
        break
    fi
done
```

## Customizing Themes

You can create your own custom theme or modify existing ones:

```toml custom-theme.toml
# Custom Alacritty Theme

[colors.primary]
background = "#1a1b26"
foreground = "#c0caf5"

[colors.normal]
black   = "#15161e"
red     = "#f7768e"
green   = "#9ece6a"
yellow  = "#e0af68"
blue    = "#7aa2f7"
magenta = "#bb9af7"
cyan    = "#7dcfff"
white   = "#a9b1d6"

[colors.bright]
black   = "#414868"
red     = "#f7768e"
green   = "#9ece6a"
yellow  = "#e0af68"
blue    = "#7aa2f7"
magenta = "#bb9af7"
cyan    = "#7dcfff"
white   = "#c0caf5"
```

<Warning>
  Always backup your current theme configuration before making custom modifications.
</Warning>

## Finding the Perfect Theme

<Steps>
  <Step title="Consider Your Environment">
    - **Bright rooms**: Use light themes (GitHub Light, Solarized Light)
    - **Dark rooms**: Use dark themes (Tokyo Night, Nord, Dracula)
    - **Mixed lighting**: Use medium-contrast themes (Gruvbox, One Dark)
  </Step>
  
  <Step title="Test for Eye Comfort">
    Use a theme for at least 30 minutes of actual work to evaluate comfort and readability.
  </Step>
  
  <Step title="Match Your Editor">
    Choose a theme that matches your code editor for visual consistency across your workflow.
  </Step>
  
  <Step title="Consider Color Blindness">
    If needed, use accessible themes like `github_light_colorblind.toml` or `github_dark_colorblind.toml`.
  </Step>
</Steps>
