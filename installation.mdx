## Prerequisites

Before installing these configurations, ensure you have the following installed on your macOS system:

<AccordionGroup>
  <Accordion title="Required Software">
    - **Neovim** (v0.9.0 or higher): `brew install neovim`
    - **Tmux**: `brew install tmux`
    - **Alacritty**: `brew install --cask alacritty`
    - **AeroSpace**: `brew install --cask nikitabobko/tap/aerospace`
    - **Git**: Pre-installed on macOS or `brew install git`
  </Accordion>

  <Accordion title="Recommended Software">
    - **0xProto Nerd Font**: `brew tap homebrew/cask-fonts && brew install --cask font-0xproto-nerd-font`
    - **TPM (Tmux Plugin Manager)**: Installed automatically in step 4
    - **Node.js**: Required for some Neovim LSP servers: `brew install node`
    - **Python 3**: For LeetCode plugin: `brew install python3`
    - **Ripgrep**: For faster searching in Neovim: `brew install ripgrep`
    - **fd**: For faster file finding: `brew install fd`
  </Accordion>
</AccordionGroup>

<Warning>
  This installation will create symlinks that replace your existing configurations. If you have existing configs, back them up first!
</Warning>

## Installation Methods

<Steps>
  <Step title="Clone the Repository">
    Clone this configuration repository to your local machine:

    ```bash
    git clone <your-repo-url> ~/dotfiles
    cd ~/dotfiles
    ```

    <Note>
      You can clone to any location, but `~/dotfiles` is a common convention. Adjust paths in subsequent steps if you choose a different location.
    </Note>
  </Step>

  <Step title="Backup Existing Configurations">
    Before proceeding, backup any existing configurations you want to preserve:

    ```bash
    # Backup existing configs
    mkdir -p ~/.config-backup
    [ -d ~/.config/nvim ] && cp -r ~/.config/nvim ~/.config-backup/
    [ -d ~/.config/alacritty ] && cp -r ~/.config/alacritty ~/.config-backup/
    [ -f ~/.tmux.conf ] && cp ~/.tmux.conf ~/.config-backup/
    [ -f ~/.aerospace.toml ] && cp ~/.aerospace.toml ~/.config-backup/
    ```
  </Step>

  <Step title="Install Neovim Configuration">
    Create a symlink for the Neovim configuration:

    ```bash
    # Remove existing config if present
    rm -rf ~/.config/nvim
    
    # Create symlink to Neovim config
    ln -s ~/dotfiles/nvim ~/.config/nvim
    ```

    LazyVim will automatically install all plugins on first launch. Start Neovim to trigger the installation:

    ```bash
    nvim
    ```

    <Info>
      First launch will take a few minutes as LazyVim downloads and installs all plugins. You'll see a dashboard with installation progress.
    </Info>

    ### Neovim Post-Installation

    After plugins are installed, you may want to:

    1. **Configure Obsidian vault path**: Edit `~/.config/nvim/lua/plugins/obsidian.lua` and update the `path` to your Obsidian vault location
    2. **Install LSP servers**: Run `:Mason` in Neovim and install language servers for your languages
    3. **Configure LeetCode**: Run `:Leet` to set up your LeetCode credentials
  </Step>

  <Step title="Install Tmux Configuration">
    Create a symlink for the Tmux configuration:

    ```bash
    # Remove existing config if present
    rm -f ~/.tmux.conf
    
    # Create symlink to Tmux config
    ln -s ~/dotfiles/tmux/tmux.conf ~/.tmux.conf
    ```

    Install Tmux Plugin Manager (TPM) and plugins:

    ```bash
    # Clone TPM
    git clone https://github.com/tmux-plugins/tpm ~/.tmux/plugins/tpm
    
    # Start Tmux
    tmux
    
    # Inside Tmux, press Ctrl+Space then I (capital i) to install plugins
    ```

    <Tip>
      The Tmux prefix is set to `Ctrl+Space`. Press `Ctrl+Space` then `?` to see all available keybindings.
    </Tip>
  </Step>

  <Step title="Install Alacritty Configuration">
    Create a symlink for the Alacritty configuration:

    ```bash
    # Remove existing config if present
    rm -rf ~/.config/alacritty
    
    # Create symlink to Alacritty config
    ln -s ~/dotfiles/alacritty ~/.config/alacritty
    ```

    The configuration references a custom Cool Night theme. If you want to use a different theme:

    ```bash
    # Edit the config to use a different theme
    nvim ~/.config/alacritty/alacritty.toml
    
    # Change the import line to any theme in themes/themes/ directory
    # For example:
    # general.import = ["~/.config/alacritty/themes/themes/tokyo_night.toml"]
    ```

    <Info>
      Alacritty configurations reload automatically when you save changes. No need to restart the terminal.
    </Info>
  </Step>

  <Step title="Install AeroSpace Configuration">
    Create a symlink for the AeroSpace configuration:

    ```bash
    # Remove existing config if present
    rm -f ~/.aerospace.toml
    
    # Create symlink to AeroSpace config
    ln -s ~/dotfiles/aerospace/aerospace.toml ~/.aerospace.toml
    ```

    Start or restart AeroSpace:

    ```bash
    # Kill existing AeroSpace instance if running
    killall AeroSpace 2>/dev/null
    
    # Start AeroSpace
    open -a AeroSpace
    ```

    <Tip>
      AeroSpace is configured to start automatically on login. You can disable this by editing `~/.aerospace.toml` and setting `start-at-login = false`.
    </Tip>

    ### AeroSpace Quick Reference

    Essential keybindings to get started:

    - `Alt+h/j/k/l`: Focus window left/down/up/right
    - `Alt+Shift+h/j/k/l`: Move window left/down/up/right
    - `Alt+1-9`: Switch to workspace 1-9
    - `Alt+Shift+1-9`: Move window to workspace 1-9
    - `Alt+/`: Toggle between horizontal and vertical tile layout
    - `Alt+Shift+;`: Enter service mode (then press `Esc` to reload config, `r` to reset layout, `f` to toggle floating)
  </Step>
</Steps>

## Alternative: Automated Installation Script

<CodeGroup>
```bash install.sh
#!/bin/bash
# Automated installation script for dotfiles

set -e

DOTFILES_DIR="$HOME/dotfiles"
BACKUP_DIR="$HOME/.config-backup"

echo "Starting dotfiles installation..."

# Create backup directory
mkdir -p "$BACKUP_DIR"

# Backup existing configs
echo "Backing up existing configurations..."
[ -d ~/.config/nvim ] && cp -r ~/.config/nvim "$BACKUP_DIR/"
[ -d ~/.config/alacritty ] && cp -r ~/.config/alacritty "$BACKUP_DIR/"
[ -f ~/.tmux.conf ] && cp ~/.tmux.conf "$BACKUP_DIR/"
[ -f ~/.aerospace.toml ] && cp ~/.aerospace.toml "$BACKUP_DIR/"

# Remove existing configs
echo "Removing existing configurations..."
rm -rf ~/.config/nvim
rm -rf ~/.config/alacritty
rm -f ~/.tmux.conf
rm -f ~/.aerospace.toml

# Create symlinks
echo "Creating symlinks..."
ln -s "$DOTFILES_DIR/nvim" ~/.config/nvim
ln -s "$DOTFILES_DIR/alacritty" ~/.config/alacritty
ln -s "$DOTFILES_DIR/tmux/tmux.conf" ~/.tmux.conf
ln -s "$DOTFILES_DIR/aerospace/aerospace.toml" ~/.aerospace.toml

# Install TPM if not present
if [ ! -d ~/.tmux/plugins/tpm ]; then
  echo "Installing Tmux Plugin Manager..."
  git clone https://github.com/tmux-plugins/tpm ~/.tmux/plugins/tpm
fi

echo "Installation complete!"
echo "Please:"
echo "  1. Launch Neovim to install plugins: nvim"
echo "  2. Start Tmux and press Ctrl+Space then I to install plugins"
echo "  3. Restart Alacritty to apply new configuration"
echo "  4. Start AeroSpace: open -a AeroSpace"
```

```bash Usage
# Make the script executable
chmod +x install.sh

# Run the installation
./install.sh
```
</CodeGroup>

## Verification

After installation, verify everything is working correctly:

<Tabs>
  <Tab title="Neovim">
    ```bash
    # Launch Neovim
    nvim
    
    # Inside Neovim, check plugin status
    :Lazy
    
    # Verify colorscheme
    :colorscheme
    # Should output: onedark
    
    # Check health
    :checkhealth
    ```
  </Tab>

  <Tab title="Tmux">
    ```bash
    # Start Tmux
    tmux
    
    # Verify plugins are installed
    # Press Ctrl+Space then type:
    # :show-environment
    
    # Check if Catppuccin theme is active
    # You should see a colorful status bar at the bottom
    ```
  </Tab>

  <Tab title="Alacritty">
    ```bash
    # Launch Alacritty
    open -a Alacritty
    
    # Verify font
    # Text should appear in 0xProto Nerd Font
    
    # Check opacity and blur
    # Window should have 80% opacity with blur effect
    ```
  </Tab>

  <Tab title="AeroSpace">
    ```bash
    # Check if AeroSpace is running
    pgrep -l AeroSpace
    
    # Test keybindings
    # Press Alt+1 to switch to workspace 1
    # Press Alt+h/j/k/l to move focus between windows
    
    # Verify config is loaded
    aerospace list-workspaces
    ```
  </Tab>
</Tabs>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Neovim plugins not loading">
    Try removing the plugin cache and reinstalling:

    ```bash
    rm -rf ~/.local/share/nvim
    rm -rf ~/.local/state/nvim
    nvim
    ```

    Then run `:Lazy sync` inside Neovim.
  </Accordion>

  <Accordion title="Tmux theme not appearing">
    Ensure TPM is installed and plugins are loaded:

    ```bash
    # Check if TPM exists
    ls ~/.tmux/plugins/tpm
    
    # Inside Tmux, reinstall plugins
    # Press Ctrl+Space then I (capital i)
    ```
  </Accordion>

  <Accordion title="Alacritty font not rendering correctly">
    Verify the Nerd Font is installed:

    ```bash
    # List installed fonts
    fc-list | grep -i "0xProto"
    
    # If not found, install it
    brew install --cask font-0xproto-nerd-font
    ```
  </Accordion>

  <Accordion title="AeroSpace keybindings not working">
    Ensure AeroSpace has accessibility permissions:

    1. Open System Settings > Privacy & Security > Accessibility
    2. Add AeroSpace to the list of allowed apps
    3. Restart AeroSpace

    ```bash
    killall AeroSpace
    open -a AeroSpace
    ```
  </Accordion>

  <Accordion title="Obsidian integration not working in Neovim">
    Update the vault path in the Obsidian plugin configuration:

    ```bash
    nvim ~/.config/nvim/lua/plugins/obsidian.lua
    ```

    Change line 14 to point to your Obsidian vault:
    ```lua
    path = "/path/to/your/obsidian/vault",
    ```
  </Accordion>
</AccordionGroup>
