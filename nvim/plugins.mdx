## Obsidian.nvim

The Obsidian plugin provides seamless integration with Obsidian vaults, enabling note-taking, linking, and daily notes directly from Neovim.

**Configuration:** `nvim/lua/plugins/obsidian.lua:1-36`

<AccordionGroup>
  <Accordion title="Full Configuration">
    ```lua
    return {
      "epwalsh/obsidian.nvim",
      version = "*",
      lazy = true,
      ft = "markdown",
      dependencies = {
        "nvim-lua/plenary.nvim",
        "hrsh7th/nvim-cmp",
      },
      opts = {
        workspaces = {
          {
            name = "personal",
            path = "/Users/leylandyang/Library/Mobile Documents/iCloud~md~obsidian/Documents/Sharedvault",
          },
        },
        completion = {
          nvim_cmp = true,
          min_chars = 2,
        },
        new_notes_location = "notes_subdir",
        daily_notes = {
          folder = "/daily notes",
          date_format = "%m-%d-%Y",
          alias_format = "%B %-d, %Y",
          default_tags = { "daily-notes" },
          template = nil,
        },
      },
    }
    ```
  </Accordion>
  
  <Accordion title="Key Features">
    - **Workspace Integration**: Configured for an iCloud-synced Obsidian vault
    - **Completion Support**: nvim-cmp integration with minimum 2 characters
    - **Daily Notes**: Automatic daily note creation with custom date formatting
    - **Note Location**: New notes are created in subdirectories for organization
    - **Lazy Loading**: Only loads when editing markdown files
  </Accordion>
  
  <Accordion title="Usage">
    The plugin automatically activates when you open markdown files. Key commands include:
    
    - Create new notes with wiki-style links
    - Navigate between notes using `gf` on links
    - Create daily notes with the configured date format
    - Autocomplete note names and tags
    
    <Warning>
    You'll need to update the `workspaces.path` to point to your own Obsidian vault location.
    </Warning>
  </Accordion>
</AccordionGroup>

## Tab Completion

Custom Tab key behavior for more intuitive completion selection.

**Configuration:** `nvim/lua/plugins/tab-complete.lua:1-40`

<AccordionGroup>
  <Accordion title="Full Configuration">
    ```lua
    return {
      {
        "L3MON4D3/LuaSnip",
        keys = function()
          return {}
        end,
      },
      {
        "hrsh7th/nvim-cmp",
        dependencies = {
          "hrsh7th/cmp-emoji",
        },
        opts = function(_, opts)
          local cmp = require("cmp")

          opts.mapping = vim.tbl_extend("force", opts.mapping, {
            ["<Tab>"] = cmp.mapping(function(fallback)
              if cmp.visible() then
                local entry = cmp.get_selected_entry()
                if not entry then
                  cmp.select_next_item({ behavior = cmp.SelectBehavior.Select })
                else
                  cmp.confirm()
                end
              else
                fallback()
              end
            end, { "i", "s" }),
            ["<CR>"] = cmp.mapping(function(fallback)
              cmp.abort()
              fallback()
            end, { "i", "s", "c" }),
          })
        end,
      },
    }
    ```
  </Accordion>
  
  <Accordion title="Behavior Explanation">
    ### Tab Key Behavior
    
    When the completion menu is visible:
    1. If no item is selected, Tab selects the first item
    2. If an item is already selected, Tab confirms it
    3. If the menu is not visible, Tab works normally
    
    ### Enter Key Behavior
    
    The Enter key is modified to:
    - Abort the completion menu
    - Insert a newline (normal behavior)
    
    This prevents accidental completion confirmations when pressing Enter.
  </Accordion>
  
  <Accordion title="Additional Features">
    - **Emoji Support**: Adds `cmp-emoji` for emoji completions
    - **LuaSnip Integration**: Disables default LuaSnip keybindings to avoid conflicts
    - **Custom Behavior**: More predictable Tab completion workflow
    
    <Tip>
    This configuration is based on a [LazyVim discussion](https://github.com/LazyVim/LazyVim/discussions/250) about improving completion UX.
    </Tip>
  </Accordion>
</AccordionGroup>

## Vim Be Good

A Neovim plugin that gamifies learning Vim motions.

**Configuration:** `nvim/lua/plugins/vimbegood.lua:1-8`

```lua
return {
  {
    "ThePrimeagen/vim-be-good",
    cmd = "VimBeGood",
    lazy = true,
  },
}
```

### Usage

Start the game with:
```vim
:VimBeGood
```

<Info>
This plugin is lazy-loaded and only activates when you run the `:VimBeGood` command, keeping startup time fast.
</Info>

### Training Modes

The plugin includes several training exercises:
- **Relative line navigation** with `j` and `k`
- **Word hopping** with `w`, `b`, and `e`
- **Find character** with `f` and `t`
- **Delete practice** with `d` motions

## Adding New Plugins

To add your own plugins, create a new file in `nvim/lua/plugins/` with this structure:

```lua
return {
  {
    "author/plugin-name",
    dependencies = { "required/dependency" },
    opts = {
      -- configuration here
    },
  },
}
```

<Steps>
  <Step title="Create plugin file">
    Create a new `.lua` file in `nvim/lua/plugins/` with a descriptive name
  </Step>
  
  <Step title="Define plugin spec">
    Return a table with the plugin specification following lazy.nvim syntax
  </Step>
  
  <Step title="Restart Neovim">
    Restart Neovim and the plugin will be automatically loaded and installed
  </Step>
  
  <Step title="Check installation">
    Run `:Lazy` to see all installed plugins and their status
  </Step>
</Steps>

<Tip>
Refer to the [example.lua](nvim/lua/plugins/example.lua) file for comprehensive examples of different plugin configuration patterns.
</Tip>
