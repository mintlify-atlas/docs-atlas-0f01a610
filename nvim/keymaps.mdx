This configuration uses primarily the default LazyVim keybindings with minimal custom modifications.

## Custom Keymaps

The custom keymaps file is intentionally minimal, relying on LazyVim's well-designed defaults.

**Configuration:** `nvim/lua/config/keymaps.lua:1-4`

```lua
-- Keymaps are automatically loaded on the VeryLazy event
-- Default keymaps that are always set: https://github.com/LazyVim/LazyVim/blob/main/lua/lazyvim/config/keymaps.lua
-- Add any additional keymaps here
```

<Note>
No custom keymaps are currently defined. All keybindings come from LazyVim's defaults.
</Note>

## Adding Custom Keymaps

To add your own keybindings, edit `nvim/lua/config/keymaps.lua`:

<Tabs>
  <Tab title="Basic Keymap">
    ```lua
    -- Map jk to escape in insert mode
    vim.keymap.set("i", "jk", "<Esc>", { desc = "Exit insert mode" })
    ```
  </Tab>
  
  <Tab title="Leader Keymap">
    ```lua
    -- Add a custom leader command
    vim.keymap.set("n", "<leader>ch", ":checkhealth<CR>", { 
      desc = "Check health" 
    })
    ```
  </Tab>
  
  <Tab title="Function Keymap">
    ```lua
    -- Map a key to a Lua function
    vim.keymap.set("n", "<leader>rr", function()
      vim.cmd("source %")
      print("Config reloaded!")
    end, { desc = "Reload config" })
    ```
  </Tab>
  
  <Tab title="Buffer-Local Keymap">
    ```lua
    -- Add keymap only for specific file types
    vim.api.nvim_create_autocmd("FileType", {
      pattern = "python",
      callback = function()
        vim.keymap.set("n", "<leader>rp", ":!python %<CR>", {
          buffer = true,
          desc = "Run Python file"
        })
      end,
    })
    ```
  </Tab>
</Tabs>

## Keymap Tips

<AccordionGroup>
  <Accordion title="Viewing All Keymaps">
    Use the built-in `which-key` plugin to explore available keymaps:
    
    - Press `<Space>` and wait to see leader key options
    - Press `g` in normal mode to see g-prefixed commands
    - Press `]` or `[` to see navigation commands
    - Run `:Telescope keymaps` to search all keymaps
  </Accordion>
  
  <Accordion title="Disabling Default Keymaps">
    To disable a LazyVim default keymap:
    
    ```lua
    -- Delete a keymap
    vim.keymap.del("n", "<leader>bd")
    
    -- Or remap it to <Nop>
    vim.keymap.set("n", "<leader>bd", "<Nop>")
    ```
  </Accordion>
  
  <Accordion title="Keymap Options">
    Common options for `vim.keymap.set()`:
    
    ```lua
    vim.keymap.set("n", "<leader>x", ":!chmod +x %<CR>", {
      desc = "Make file executable",  -- Description for which-key
      silent = true,                   -- Don't show command in command line
      noremap = true,                  -- Don't remap recursively (default true)
      buffer = 0,                      -- Only apply to current buffer
      expr = false,                    -- Not an expression mapping
    })
    ```
  </Accordion>
</AccordionGroup>

<Warning>
Be careful not to override essential Vim keybindings. Check if a key is already mapped with `:map <key>` before assigning it.
</Warning>

## Learning LazyVim Keymaps

<Steps>
  <Step title="Use Which-Key">
    Press `<Space>` and wait 1 second. A popup will show all available leader commands.
  </Step>
  
  <Step title="Read the Docs">
    Visit [LazyVim keymaps documentation](https://lazyvim.github.io/keymaps) for a complete reference.
  </Step>
  
  <Step title="Explore with Telescope">
    Run `:Telescope keymaps` to search through all available keybindings.
  </Step>
  
  <Step title="Practice Regularly">
    Use `:VimBeGood` to practice Vim motions and improve muscle memory.
  </Step>
</Steps>
